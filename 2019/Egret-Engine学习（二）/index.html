Egret Engine学习（二）

发表于 2019-05-16 | 分类于 Egret

简介

  前面学习了Egret Engine的安装和第一个项目的创建、调试运行，现在来学习一下项目目录下的配置文件。

index.html

  index.html为项目的入口文件，可以根据项目需求进行修改。其中body标签包含div和script标签。

data-entry-class：文件类名称，也是项目的入口类。
data-orientation：旋转模式。
data-scale-mode：适配模式。
data-frame-rate：帧频数。
data-content-width：游戏内舞台的宽。
data-content-height：游戏内舞台的高。
data-multi-fingered：多指最大数量。
data-show-fps：是否显示 fps 帧频信息。
data-show-log：是否显示 egret.log 的输出信息。
data-show-fps-style：fps面板的样式。支持5种属性，x:0, y:0, size:12, textColor:0xffffff, bgAlpha:0.9

script执行逻辑是按manifest.json里配置的顺序加载js脚本，所有脚本加载完后，回调到callback函数。

callback函数参数是一个对象，包括以下3个可选属性：
renderMode”: 引擎渲染模式，”canvas” 或者 “webgl”;
audioType”: 使用的音频类型，0:默认，2:web audio，3:audio;
calculateCanvasScaleFactor”：屏幕的物理像素适配方法，使用默认的即可。

manifest.json

  项目入口文件加载的js脚本配置，包含initial和game两个配置。initial是基础库的js脚本，game是用户编写的项目js脚本。

tsconfig.json

   Typescript项目的配置文件。TypeScript编译器编译代码之前，会首先读取这个配置文件，并根据其中的属性来设置 TypeScript 项目的编译参数。

compilerOptions字段：
target:编译之后生成的JavaScript文件需要遵循的标准，默认为 es5，兼容性比较好，不建议修改；
outDir:编译出来的js文件，放到哪个目录下，默认编译到bin-debug里，目前暂不支持修改；
experimentalDecorators:启用实验性质的语法装饰器，引擎里的部分库使用了最新的语法，需要开启这个配置；
lib: 编译需要的库文件，默认有3个，你可以根据需求自行添加；
其他常用参数
“sourceMap”: true:把.ts 文件编译成.js 文件时，生成对应的 .js.map 文件，该文件可以让用户直接在浏览器里调试 ts 文件
“removeComments”: true: 编译 .js 时删除原本 .ts 文件中的注释。
include 字段,表示哪些文件会参与编译。
更详细的编译参数，可以参考tsconfig.json 的官方文档。

egretProperties.json

  项目配置文件。包含引擎版本信息，模块信息等。其中template字段如果存在，那么在发布Html5项目时，会使用template/web/index.html来作为入口文件。

modules 字段可以定义项目中引用的所有库文件。每一个库都是形如 { “name”:”moduleName” , “path”:”modulePath”} 的配置格式。name 字段是库名，path 字段是库文件存放路径。如果在库项目中，项目运行时直接加载此路径所对应的库；如果在项目外，引擎编译时会首先将此路径所对应的库拷贝至项目中的libs/modules文件夹中，然后加载该文件夹中的库。

.wing/launch.json

  调试运行的运行环境配置。如入口文件，运行端口号信息等。

.wing/tasks.json

  配置任务运行的行为方式。如launch.json配置里用到的build任务。

总结

  项目通过json格式的配置文件来控制编译，运行等各种环境和入口。理解这些配置文件的作用有助于理解项目的运行的逻辑和过程。
